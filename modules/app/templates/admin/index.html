Welcome to the MiKTMC User Portal

The MiKTMC User Portal provides authorization and access for users of MiKTMC applications. It is intended to be used alongside Shibboleth, which provides the authentication and login.

Getting Access to the User Portal

Contact miktmc-devs@umich.edu to have someone granted access to the User Portal.

Basic Usage

Follow these instructions for users whose institutions (usually academic) allow federated login via InCommon/Shibboleth. For users with University of Michigan Friend Accounts, Google, or Microsoft accounts, see instructions below.

  1. Click "Organizations" and make sure the user's organization is in the list. If not, add it. NOTE: this does not add the organization to the initial login page. Contact miktmc-devs@umich.edu to add an organization to login page dropdown list.
  2. Click Users -> Create
  3. Enter the user's email address as the Shibboleth ID.
  4. Enter First Name and Last Name and choose the organization. Any additional user information is optional.
  5. Select "uploader_user" as the Group.
  6. Click "Save"

Users with University of Michigan Friend Accounts

A University of Michigan Friend Account is a good choice for those users without an institutional Shibboleth account and don't have or want to use a Google or Microsoft account.

  1. Have the user create a friend account here: https://friend.weblogin.umich.edu/friend/ and provide you with the email address they used to register.
  2. Follow the Basic Usage Instructions above, entering the Shibboleth ID in the following format: [fullemailaddresswithdomain]@umich.edu, leaving out the @ symbols. So jane.smith@university.edu would become jane.smithuniversity.edu@umich.edu
  3. When the user logs in to the Uploader, they should choose “University of Michigan” and use their full email address as the username. Example: jane.smith@university.edu

Users with Google or Microsoft Accounts

Users logging in with Google or Microsoft accounts require an additional step because the User Portal requires the internal ID of the user's account.

  1. Have the user choose the Google or Microsoft logo on the login page and log in with their account.
  2. If the user is able to log in successfully, they should see the Upload page with a message that they are not authorized. Have the user note the time/date.
  3. Check for an email notification sent to the MiKTMC Uploader administrator list at that time/date that contains the user's internal user ID, usually in the format "XXXXXXXXXXXXXX@google.com" for a Google account or "XXXXXX-XXXXXX-XXXXX@live.com" for a Microsoft Account.
  4. Follow the Basic Usage instructions above and use the ID from the notification email as the Shibboleth ID.

Administrator Access to the Uploader

Admin privileges should not be granted without approval from a Project Manager stating why this person should have administrative access to the Uploader. MiKTMC Uploader admins can edit all uploads, lock uploads, and see a link to view the files in an upload. Follow the instructions above but choose "uploader_admin" as the Group. If the user already exists, find the user in the user list and edit their record to change the group. If they need to view the files, also contact miktmc-devs@umich.edu to grant them the appropriate permissions.
